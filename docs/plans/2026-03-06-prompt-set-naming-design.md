# Prompt Set Naming Design

## Context
Prompt sets generated in the tool only have an ID (`promptSetID`). We need to allow these prompt sets to be named.

## Requirements
1. **New Prompt Sets:** A `promptSetName` parameter will be auto-generated based on the original `prompt` (first 3-5 words) when the user leaves the input blank.
2. **UI Override:** `SettingsSection.tsx` will display the auto-generated prompt set name to the user before generation. The user can override this value before or after generation.
3. **Existing Prompt Sets:** A database migration script will be built to backfill existing sets with an autogenerated name.
4. **Gallery Management:** The gallery sidebar (`PromptSetIDManager.tsx`) will be adapted to easily rename existing sets.
5. **Community Integration:** `promptSetName` will also be captured in the `LeagueEntry`/`CommunityEntry` schemas to maintain consistency.

## Architecture & Data Model

### `src/lib/types.ts`
We will add the `promptSetName` string correctly mapped out into `GeneratedImage`, `GenerationSettings`, and `CommunityEntry` structures. 

```typescript
export interface GenerationSettings {
    ...
    promptSetID?: string;
    promptSetName?: string; // NEW
}

export interface GeneratedImage {
    ...
    promptSetID?: string;
    promptSetName?: string; // NEW
}

export interface CommunityEntry {
    ...
    promptSetID?: string;
    promptSetName?: string; // NEW
}
```

### Endpoints
* `src/app/api/generate/nanobanana/route.ts`:
  Will generate a name if none is explicitly provided by inspecting the subject/modifiers to pull a substring.
* `src/app/api/generate/draft/route.ts`:
  Will be updated to support saving and reading the `promptSetName`.
* `src/hooks/useImageDetails.ts` & `src/app/gallery/useGallery.ts`:
  Will be updated for the new Firestore field update actions allowing `promptSetName` changes to sync.

## Migration Strategy
A node script will be made `scripts/migrate_prompt_set_names.ts`.
This will run through Firebase admin fetching `GeneratedImage` documents from `/users/{uid}/images` and `/leagueEntries/`. Docs missing a `promptSetName` will take a `substring` and `titlecase` of the `prompt` associated with it.
